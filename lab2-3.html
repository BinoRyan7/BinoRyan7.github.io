1. Define the Problem
    • This step invloves gathaering the requirements of the program needed to solve the problem.
    • Questions that should be answered:
      1. What data do we need to process?
      2. How should we process the data?
      3. What output should the program deliver?
      4. How shuld the user interface look?
  2. Design the Program
    • Program design focuses on breaking down the program into smaller more managable components. Various tools are used to help design a program.
    • Commonly used tool:
      • Structure Charts
      • Algorithim
      • Flow Charts
      • Descision Tables
      • Pseudocode
  3. Code the program
    • Once the program has been designed, a developer then writes the code.
    • Coding often involves translating the design from charts and pseudocode into a specific programming language. Usually the language is chosen during the "defing" stage.
  4. Testing and Debugging
    • After code has been written this stage revolves around fixing sny bugs in the program.
    • There are two kinds of erros that cause bugs:
      1. Syntax Errors: These are like grammatical mistakes in the code.
      2. Logical Errors: These mistakes might cause an other than intended output but th ecode still works.
    • The programmer must fix these mistakes by carefully examining the alreaday written code for errors.
  5. Documentation
    • After coding has been completed the programmer must write a manual that generally contains the following:
      1. An overview of the programs functionality
      2. In-depth expanations of program features.
      3. Documentation of all commands
      4. A thorough description of error messages generated by the program.
  6. Deploying and Maintaining the Program
    • A software team will be responsible lfor fixing new errors and updating the software with new features.
    • When a new feature should be added to the program. cycle restarts as developers begin to define what problem the new feature will solve.
